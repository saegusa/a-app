# 将棋アプリ MVP アーキテクチャ（Web / Angular v21）

## 目的
- Webブラウザ上で、通常将棋の CPU 対戦 MVP を提供する。
- 対局体験を優先し、CPU は「ランダム合法手」で構築する。

## スコープ（MVP）
- 9x9 盤面表示
- 初期配置
- 手番管理（先手: プレイヤー、後手: CPU）
- 駒移動（合法手ベース）
- 成り（簡易）
- 持ち駒、打ち（同筋歩チェックのみ）
- 王の捕獲で勝敗決定

## 構成
- `src/app/app.component.*`
  - 盤面 UI、クリック操作、ログ表示
  - CPU 手番のトリガー
- `src/lib/shogi.ts`
  - ドメインモデル（駒、手、局面）
  - 初期局面生成
  - 合法手生成（各駒ごとの移動）
  - 局面更新
  - CPU の手選択（ランダム）

## 制約と今後
- 千日手、王手放置、打ち歩詰めの完全対応は未実装
- 強さ向上は評価関数・探索導入で拡張予定
